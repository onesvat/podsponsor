**Podsponsor** automatically finds and removes sponsor ads, promotions, and repetitive intros/outros from your downloaded podcast files. You simply point it at a folder of podcast episodes, and it seamlessly cuts out the adsтАФgiving you clean, uninterrupted listening files. тЬи

## ЁЯза Why It's Better (The Technical Details)
Unlike traditional tools that rely on crowdsourced databases (like SponsorBlock) or simple acoustic jingle detection, Podsponsor uses a **Two-Phase Multi-Episode Analysis** approach driven by LLMs. By comparing the transcripts of multiple episodes of the same podcast against each other, it can mathematically detect identical, pre-recorded sponsor segments. It feeds these high-probability cross-matches into an LLM (like OpenAI or Moonshot) for final validation, ensuring incredible accuracy at cutting out ads while preserving organic host content.

## тЪЩя╕П How It Works

Podsponsor abandons single-pass processing for a smarter, two-phase pipeline:

1. **Phase 1: Batch Transcription & Cross-Matching**
   - Transcribes all your un-processed podcast episodes in a folder using `whisperx`.
   - Stores transcript segments directly in `<episode>.podsponsor.json`.
   - Scans all transcripts against each other using fuzzy string matching. If the exact same paragraph appears in Episode 12 and Episode 40, it is flagged as `[REPEATED]`.

2. **Phase 2: LLM Validation & Cutting**
   - The transcript, enhanced with `[REPEATED]` and `[HIGH_FREQ]` markers, is sent to an LLM via strict JSON structured output (`response_format`).
   - The LLM easily identifies the exact start and end segments of the ad blocks, and also writes a summary of the episode.
   - `ffmpeg` surgically removes the ad blocks based on segment timings.
   - The algorithm mathematically shifts timestamps in your `.srt` file so it remains synced with the cleaned audio.

## тЬи Features
- **GPU-Accelerated Transcription тЪб**: Uses `whisperx` for fast transcription and segment-level timing data.
- **Transparent Ad Signals ЁЯзн**: LLM detection uses repeated-content markers and high-frequency hints to improve sponsor block boundary decisions.
- **Fail-Safe Processing ЁЯЫбя╕П**: Original `.mp3` files are safely saved before cuts are applied (backup location is configurable).
- **Multi-Provider LLM Fallback ЁЯФД**: Define a list of API providers (OpenAI, Ollama, synthetic.new, etc). If one hits a rate limit or times out, Podsponsor instantly fails over to the next provider.
- **Automatic Summarization ЁЯУЭ**: Generates a beautiful `.md` summary of the podcast episode in your language of choice.
- **Resume Capability ЁЯТ╛**: Tracks processing state per episode in `<episode>.podsponsor.json`.


   *Note: Ensure `ffmpeg` is installed on your system (`sudo apt install ffmpeg` or `brew install ffmpeg`).*

   ```bash

   ```

3. Add your LLM API keys to `config.yaml`. ЁЯФС

## ЁЯТ╗ Usage

Run Podsponsor by simply pointing it at a folder containing your `.mp3` podcast episodes. This is designed to point directly to the output folders generated by downloaders like `audiobookshelf`:

```bash
python podsponsor.py /path/to/audiobookshelf/podcasts/folder/
```

### Configuration Options (`config.yaml`)

- **`whisper`**: Define the model size (`medium`, `large-v2`), device (`cuda`, `cpu`), compute type (`float16`), and optional `chunk_size` (default `20`) to bias toward shorter/longer segment chunks.
- **`llm`**: Set your `summary_language` and define your `providers` array for failover support. (Optional) override the default prompt by adding `llm.prompt`.
- **`detection`**: Tweak confidence thresholds and set guardrails (e.g., maximum allowed ad cut duration).
- **`backup`**: Set `enabled: false` to disable backups entirely, or configure `location` for where original `.mp3`/`.srt` backups are stored (relative path per episode directory or absolute path).
